# 缓存策略设计

> 目的：统一缓存的使用边界与 key 规范，避免缓存污染、雪崩、击穿及一致性问题。

---

## 1. 缓存分层
- 本地缓存（可选）：热点配置、字典（注意多实例一致性）
- 分布式缓存（Redis）：会话、权限缓存、业务热点数据

---

## 2. Key 规范（建议）
统一前缀：`zephyr:{env}:{service}:{module}:{biz}`

示例：
- `zephyr:prod:platform:auth:refresh:{userCode}`
- `zephyr:prod:platform:rbac:perm:{userCode}`

---

## 3. TTL 策略（建议）
- 权限/菜单缓存：短 TTL（例如 5~30min）+ 主动失效（角色变更时删 key）
- Refresh Token：长 TTL（例如 7d）+ 退出时删除
- 黑名单 jti：TTL = token 剩余有效期

---

## 4. 一致性策略（摘要）
优先采用：
- Cache-Aside（旁路缓存）：先读缓存，miss 读 DB，回填缓存；写 DB 后删除缓存
- 对“强一致”场景避免缓存（或引入消息驱动失效）
