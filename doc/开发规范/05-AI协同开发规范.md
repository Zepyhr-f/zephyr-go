# AI 协同开发规范

本规范介绍了后端团队如何利用 Cursor 的 AI Agent 进行高效的功能开发，通过结构化的流程，从需求定义到代码实现。

## 协同核心理念

将复杂功能的开发过程拆解为透明、可控的步骤：
1. **定义范围**：通过产品需求文档 (PRD) 明确要构建的业务逻辑。
2. **详细规划**：将 PRD 拆解为细粒度、面向 Go/go-zero 的开发任务列表。
3. **迭代实现**：引导 AI 每次只处理一个任务（如：创建 Model -> 实现 Logic -> 完善 Handler）。

## AI 开发工作流

### 第一步：创建产品需求文档 (PRD)
首先为功能制定蓝图。PRD 应明确我们要构建什么、涉及哪些表结构、以及核心业务规则。

在 Cursor Agent 中输入：
```text
使用 @create-prd.mdc
我想开发的后端功能详细描述如下：[描述你的功能，例如：实现一个订单导出接口]
参考这些文件：[@00-建表规范.md @base.go]
```

### 第二步：从 PRD 生成任务列表
有了 PRD 后，下一步是为后端开发任务生成详细步骤。

在 Cursor Agent 中输入：
```text
现在基于 @MyFeature-PRD.md 使用 @generate-tasks.mdc 生成后端任务列表
```

### 第三步：审查任务列表
AI 会生成包含 Model、Logic、Handler 等各层的开发任务。检查任务顺序是否符合依赖关系（通常先做底层，再做上层）。

### 第四步：引导 AI 执行任务
使用任务处理指令引导 AI 逐步完成编码与调试。

在 Cursor Agent 中从第一个任务开始执行：
```text
请开始执行任务 1.1 并使用 @process-task-list.mdc
```
*注意：只需在第一个任务时引用规范，后续 AI 会根据反馈自动流转。*

### 第五步：审查、批准并推进
当 AI 完成每个任务（例如生成了结构体或 Logic 逻辑）时：
* **如果变更符合规范**：回复“yes”，指示 AI 标记完成并进入下一个任务。
* **如果需要修改**：提供具体的 Go 规范反馈（如：修改结构体标签、修复 JSON Tag），修正当前的实现。

## AI 协同优势

*   **结构化开发**：确保后端各层（Model/Logic/Handler）的代码一致性。
*   **逐步验证**：可以逐层单元测试或审查代码，降低大面积逻辑错误的风险。
*   **管理复杂度**：将复杂的微服务交互分解为较小的编码任务。
*   **自动遵循规范**：AI 在协同过程中会引用本项目已定义的实体基类、统一返回对象等。

## 成功建议

- **提供明确的 Context**：在任务开始前，让 AI 读取相关的 `base.go`、`types.go` 或已有的相似 Logic，它能更好地模仿现有风格。
- **验证数据库变更**：AI 在修改数据库相关代码时，请务必核对生成的 SQL 与 `00-建表规范.md` 是否一致。
