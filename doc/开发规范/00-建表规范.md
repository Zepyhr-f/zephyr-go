# 数据库建表规范

> **适用范围**：Zephyr 项目所有微服务数据库表
> **强制级别**：所有新建表必须严格遵循本规范，存量表逐步对齐

---

## 一、实体基类与表字段映射

Zephyr 后端框架定义了三种实体基类，分别对应不同场景的业务表。每张业务表都必须与对应的实体基类保持字段一致。

### 1.1 基类继承关系

```
BaseEntity
├── CodeEntity  (+ code)
│   └── TreeEntity  (+ parent_code, leaf)
```

| 基类 | 适用场景 | 额外字段 |
|------|---------|---------|
| `BaseEntity` | 纯数据记录，无需对外关联 | 无 |
| `CodeEntity extends BaseEntity` | 需要对外关联的业务实体（部门、用户、角色等） | `code` |
| `TreeEntity extends CodeEntity` | 树形结构数据（部门树、菜单树等） | `parent_code`, `leaf` |

### 1.2 BaseEntity 基础字段（所有业务表必须包含）

| 字段名 | 数据类型 | 是否必须 | 默认值 | 说明 |
|--------|---------|---------|--------|------|
| `id` | `BIGINT` NOT NULL | ✅ 是 | - | 主键 ID，**雪花算法**（非自增） |
| `tenant_code` | `VARCHAR(12)` | ✅ 是 | `'000000'` | 租户编码，用于多租户数据隔离 |
| `create_user` | `VARCHAR(64)` | ✅ 是 | NULL | 创建人编码 |
| `create_time` | `DATETIME` | ✅ 是 | `CURRENT_TIMESTAMP` | 创建时间 |
| `update_user` | `VARCHAR(64)` | ✅ 是 | NULL | 最后更新人编码 |
| `update_time` | `DATETIME` | ✅ 是 | `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 最后更新时间 |
| `if_deleted` | `TINYINT(1)` NOT NULL | ✅ 是 | `0` | 逻辑删除标志：`0`=正常，`1`=已删除 |

> [!IMPORTANT]
> **`if_deleted` 字段是逻辑删除的唯一标准方式**，严禁使用物理删除（`DELETE FROM`）操作业务数据。GORM 中请通过相应的 scope 拦截。
> `id` 字段仅做数据唯一标识，**必须使用雪花算法**，禁止数据库自增（`AUTO_INCREMENT`）。涉及业务唯一性需额外创建 `code` 字段。

### 1.3 CodeEntity 扩展字段（需要对外关联的表必须包含）

| 字段名 | 数据类型 | 是否必须 | 默认值 | 说明 |
|--------|---------|---------|--------|------|
| `code` | `VARCHAR(64)` NOT NULL | ✅ 是 | - | **业务唯一编码**，对外关联的统一标识 |

> [!TIP]
> 所有需要被其他表关联的业务实体（用户、角色、部门、岗位、菜单等），**必须**包含 `code` 字段作为业务唯一标识。关联表通过 `xxx_code` 字段引用目标表的 `code`。

### 1.4 TreeEntity 扩展字段（树形结构表必须包含）

| 字段名 | 数据类型 | 是否必须 | 默认值 | 说明 |
|--------|---------|---------|--------|------|
| `parent_code` | `VARCHAR(64)` | ✅ 是 | NULL | 父节点编码，`NULL` 表示根节点 |
| `leaf` | `TINYINT(1)` | ✅ 是 | `1` | 是否叶子节点：`1`=是，`0`=否 |

### 1.5 对应后端 Go 模型

后端的数据库模型（Model）结构请参考 Go 代码 `model/base.go` 等定义。

## 二、建表 SQL 标准模板

### 2.1 BaseEntity 基础表模板（纯数据记录，无需对外关联）

```sql
CREATE TABLE `zephyr_模块_表名` (
  -- ① 主键与基础字段
  `id`          BIGINT       NOT NULL COMMENT '主键ID (雪花算法)',

  -- ② 业务核心字段
  -- ...

  -- ③ 基础审计与租户字段
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识 (0=正常 1=已删除)',

  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='【填写表注释】';
```

### 2.2 CodeEntity 编码表模板（需要对外关联的业务实体）

```sql
CREATE TABLE `zephyr_模块_表名` (
  -- ① 主键与基础字段
  `id`          BIGINT       NOT NULL COMMENT '主键ID (雪花算法)',

  -- ② 业务唯一编码（对外关联标识）
  `code`        VARCHAR(64)  NOT NULL COMMENT '业务编码 (唯一标识)',

  -- ③ 其他业务字段
  -- ...

  -- ④ 基础审计与租户字段
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识 (0=正常 1=已删除)',

  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code_del` (`code`, `if_deleted`, `tenant_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='【填写表注释】';
```

> [!IMPORTANT]
> 唯一索引必须命名为 `uk_code_del`，且**必须包含 `code`、`if_deleted`、`tenant_code` 三个字段**，以确保在多租户 + 逻辑删除场景下的业务唯一性。

### 2.3 TreeEntity 树形表模板（树形结构数据）

```sql
CREATE TABLE `zephyr_模块_表名` (
  -- ① 主键与基础字段
  `id`          BIGINT       NOT NULL COMMENT '主键ID (雪花算法)',

  -- ② 业务唯一编码 + 树形字段
  `code`        VARCHAR(64)  NOT NULL COMMENT '业务编码 (唯一标识)',
  `parent_code` VARCHAR(64)  DEFAULT NULL COMMENT '父节点编码 (NULL表示根节点)',
  `leaf`        TINYINT(1)   DEFAULT 1 COMMENT '是否叶子节点 (1=是 0=否)',

  -- ③ 其他业务字段
  -- ...

  -- ④ 基础审计与租户字段
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识 (0=正常 1=已删除)',

  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code_del` (`code`, `if_deleted`, `tenant_code`),
  INDEX `idx_parent_code` (`parent_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='【填写表注释】';
```

### 2.4 中间关联表（无需继承 BaseEntity）

中间表（如 `sys_user_role`、`sys_role_menu`）只存储关联关系，**不需要审计字段**：

```sql
CREATE TABLE `zephyr_sys_user_role` (
  `user_code` VARCHAR(64) NOT NULL COMMENT '用户编码 (关联 sys_user.code)',
  `role_code` VARCHAR(64) NOT NULL COMMENT '角色编码 (关联 sys_role.code)',
  PRIMARY KEY (`user_code`, `role_code`),
  INDEX `idx_role_code` (`role_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户和角色关联表';
```

> [!TIP]
> 中间表字段命名统一使用 `xxx_code` 格式（如 `user_code`、`role_code`），表示引用对应业务表的 `code` 字段值。

## 三、字段命名规范

### 3.1 通用命名约定

| 规则 | 说明 | 示例 |
|------|------|------|
| 全部小写 + 下划线 | 使用 snake_case 风格 | `user_name`、`dept_code` |
| 布尔/状态类字段 | 用 `status`（业务状态）或 `0/1` 枚举型 `TINYINT(1)` | `status TINYINT(1)` |
| 外键/关联字段 | 以关联表名的单数形式加 `_code` 结尾（引用对方 `code`） | `dept_code`、`role_code` |
| 时间字段 | 统一用 `_time` 结尾，类型为 `DATETIME` | `create_time`、`expire_time` |
| 金额字段 | 使用 `DECIMAL(18, 2)` 类型，单位说明写在注释 | `amount DECIMAL(18, 2) COMMENT '金额（元）'` |
| 文本类字段 | 短文本用 `VARCHAR`，长文本用 `TEXT` | `remark VARCHAR(500)`、`content TEXT` |

### 3.2 禁止事项

- ❌ 禁止使用 `MySQL` 关键字作为字段名（如 `key`、`value`、`desc`、`order`）
- ❌ 禁止使用 `is_xxx` 形式命名，改用 `xxx_flag` 或 `status`（逻辑删除统一用 `if_deleted`）
- ❌ 禁止使用 `FLOAT` / `DOUBLE` 存储金额，必须用 `DECIMAL`
- ❌ 禁止在 `VARCHAR` 字段不加长度限制
- ❌ 禁止业务表使用 `AUTO_INCREMENT`，主键统一使用雪花算法
- ❌ 禁止树形表用 `0` 或 `-1` 表示根节点，统一使用 `NULL`

## 四、索引规范

### 4.1 索引命名

| 类型 | 前缀 | 示例 |
|------|------|------|
| 普通索引 | `idx_` | `idx_username` |
| 唯一索引 | `uk_` | `uk_code_del` |
| 联合索引 | `idx_` + 多字段 | `idx_user_role` |

### 4.2 索引创建原则

- `PRIMARY KEY` 必须是 `id`，类型 `BIGINT`，**禁止自增**。
- 继承 `CodeEntity` 的表，`code` 字段必须建立唯一索引，且**必须包含 `if_deleted` 和 `tenant_code`** 以支持逻辑删除和多租户下的唯一性。
- 唯一索引命名固定为：`uk_code_del`。
- 继承 `TreeEntity` 的表，必须建立普通索引 `idx_parent_code` (`parent_code`)。
- 查询频率高的字段加普通索引 `idx_字段名`。
- 避免在低基数字段（如 `status`、`if_deleted`）上创建单独索引。

---

## 五、字符集与存储引擎规范

- **字符集**：统一使用 `utf8mb4`，支持 Emoji 及四字节字符
- **排序规则**：`utf8mb4_general_ci`（不指定时使用数据库默认值）
- **存储引擎**：统一使用 `InnoDB`，支持事务与外键

## 六、完整建表示例

### 6.1 CodeEntity 示例（岗位表）

```sql
CREATE TABLE `zephyr_sys_post` (
  `id`          BIGINT       NOT NULL COMMENT '主键ID (雪花算法)',
  `code`        VARCHAR(64)  NOT NULL COMMENT '岗位编码 (业务唯一标识)',
  `post_name`   VARCHAR(50)  NOT NULL COMMENT '岗位名称',
  `order_num`   INT          DEFAULT 0 COMMENT '显示顺序',
  `status`      TINYINT(1)   DEFAULT 1 COMMENT '状态 (1=正常 0=停用)',
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识 (0=正常 1=已删除)',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code_del` (`code`, `if_deleted`, `tenant_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='岗位信息表';
```

### 6.2 TreeEntity 示例（部门表）

```sql
CREATE TABLE `zephyr_sys_dept` (
  `id`          BIGINT       NOT NULL COMMENT '主键ID (雪花算法)',
  `code`        VARCHAR(64)  NOT NULL COMMENT '部门编码 (业务唯一标识)',
  `parent_code` VARCHAR(64)  DEFAULT NULL COMMENT '父部门编码 (NULL表示根节点)',
  `leaf`        TINYINT(1)   DEFAULT 1 COMMENT '是否叶子节点 (1=是 0=否)',
  `dept_name`   VARCHAR(50)  NOT NULL COMMENT '部门名称',
  `order_num`   INT          DEFAULT 0 COMMENT '显示顺序',
  `status`      TINYINT(1)   DEFAULT 1 COMMENT '部门状态 (1=正常 0=停用)',
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识 (0=正常 1=已删除)',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code_del` (`code`, `if_deleted`, `tenant_code`),
  INDEX `idx_parent_code` (`parent_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='部门表';
```

## 七、后端模型类对应规范

每张业务表都必须有对应的 Go Model 结构体。

> [!TIP]
> 基础字段将由 GORM 的 Hooks 机制或通用的拦截机制自动处理（如 `create_user`、`create_time` 等），无需在业务代码中手动频繁赋值。
