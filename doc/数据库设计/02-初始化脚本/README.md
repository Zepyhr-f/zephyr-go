# 4.2-初始化脚本

本目录存放“种子数据/初始化数据”SQL（DML），用于初始化字典、默认账号、默认角色菜单等基础数据。

---

## 使用约定
- 执行顺序以 `04-数据库设计/README.md` 为准
- 生产环境初始化数据需走发布流程（审核与回滚预案）

---

## 文件清单

| 文件 | 关键词（便于 AI 检索） | 说明 |
|---|---|---|
| `4.2.1-infrastructure.sql` | 初始化、infra | 基础设施模块初始化数据 |
| `4.2.2-RBAC_init.sql` | RBAC、角色、菜单、管理员 | RBAC 初始数据（默认角色/菜单等） |
| `4.2.3-sys_init.sql` | system、字典、配置 | 系统模块初始化数据 |
| `4.2.4-sys_monitor_init.sql` | monitor、监控 | 监控模块初始化数据 |
| `4.2.5-sys_infrastructure_init.sql` | infra、系统基础 | 系统基础设施初始化数据 |

## 阅读建议
- 想确认默认角色、菜单、字典和系统种子数据时，从本目录开始。
- 如果是数据库执行顺序问题，先回看上级数据库设计 README。

## 维护规则
- 新增初始化脚本时，必须同步更新本 README，并在上级入口中确认执行顺序。
- 生产环境初始化数据的变更应与发版流程保持一致，避免手工执行失控。
- 初始数据如果承载业务规则，应同步更新对应功能设计文档。
