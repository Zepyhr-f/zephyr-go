# 04-数据库设计

本目录负责管理 Zephyr 项目的数据库资产，包括建表脚本、初始化脚本与执行顺序说明。

## 子模块概览

| 目录 | 关键词（便于 AI 检索） | 说明 |
|---|---|---|
| [4.1-建表脚本](./4.1-建表脚本/) | DDL、建表、表结构、索引 | 定义系统物理表结构 |
| [4.2-初始化脚本](./4.2-初始化脚本/) | 初始化、DML、DCL、权限、字典 | 初始化环境、权限和基础数据 |

---

## 执行顺序

1. 先执行 `4.2.1-infrastructure.sql`，完成基础数据库环境准备。
2. 再按顺序执行 `4.1.x-` 系列建表脚本，建立表结构。
3. 最后执行 `4.2.2+` 系列初始化脚本，写入权限与默认数据。

## 阅读建议
- 想看表结构边界时，先进入 [4.1-建表脚本](./4.1-建表脚本/README.md)。
- 想看初始化数据和执行顺序时，进入 [4.2-初始化脚本](./4.2-初始化脚本/README.md)。

## 维护规则
- 本目录是数据库脚本的统一入口，新增 SQL 文件时必须同步更新对应子目录 README。
- 表结构、索引、初始化顺序发生变化时，必须同步更新说明文档。
- 业务字段语义、权限模型与功能规则应分别回链到功能设计或架构设计文档，不在本目录重复维护业务口径。
