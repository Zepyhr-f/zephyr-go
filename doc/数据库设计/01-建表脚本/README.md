# 4.1-建表脚本

本目录存放“结构定义类”SQL（DDL），用于创建表结构、索引与约束。

---

## 使用约定
- 执行顺序以 `04-数据库设计/README.md` 为准
- 文件名使用编号前缀，保证可排序（例如 `4.1.1-*`）

---

## 文件清单

| 文件 | 关键词（便于 AI 检索） | 说明 |
|---|---|---|
| `4.1.1-rbac_core.sql` | RBAC、user、role、menu | RBAC 核心表结构 |
| `4.1.2-security_audit.sql` | audit、log | 安全审计相关表结构 |
| `4.1.3-platform_config.sql` | dict、config | 平台配置/字典相关表结构 |
| `4.1.3-sys_job.sql` | job、scheduler | 定时任务相关表结构 |
| `4.1.4-infrastructure_module.sql` | infra、module | 基础设施模块表结构 |

## 阅读建议
- 想确认表结构边界时，从本目录开始。
- 如果问题是“为什么这样建模”，应回看架构设计或功能设计中的对应说明。

## 维护规则
- 新增或调整建表脚本时，必须同步更新本 README 和上级数据库设计入口。
- 文件名应保持编号递增，避免执行顺序和可检索性混乱。
- 表结构变更涉及字段语义时，应同步检查功能设计和数据模型文档。
